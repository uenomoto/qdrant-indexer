# サンプルドキュメント

このファイルはチャンク分割のテストに使用する。

## 背景

Megurip アプリケーションのインフラストラクチャを Terraform で管理するリポジトリである。
AWS のサーバーレスアーキテクチャ（Lambda, API Gateway, S3, CloudFront）を活用し、
コスト効率の高いインフラ構成を実現している。バックエンドは Go/Echo で実装され、
aws-lambda-go-api-proxy を使用して Lambda 上で動作する。Worker Lambda は
Python 3.13 で実装され、ルート計算や AI 最適化といった非同期処理を担当する。
データベースには Neon PostgreSQL を採用し、サーバーレスとの相性を最大限に活かしている。

## 設計方針

レゴブロック型アーキテクチャを採用する。modules/ がレゴブロック（部品）で、
environments/ が組み立て場所という明確な責務分離を実現している。
各モジュールは単一責任の原則に従い、他のモジュールの内部実装に依存しない。
モジュール間の連携は environments/ レイヤーで変数の受け渡しによって行われ、
モジュール自体は独立してテスト可能である。この設計により、新しい環境の追加や
既存環境の変更が容易になり、インフラの変更に対するリスクを最小限に抑えている。

## コードブロック内のテスト

以下のコードブロック内の `##` で分割してはいけない。コードブロックの検出は
バッククォート3つの開始・終了をトラッキングすることで実現する。
ネストされたコードブロックや異なる言語指定にも対応する必要がある。

```markdown
## これは分割しない
この行もコードブロック内なので無視する。
## これも分割しない
```

コードブロックの後のテキスト。上記のコードブロック内にある `##` は
チャンク分割のトリガーとして認識されてはならない。この検証は
セマンティック検索の品質に直結する重要なテストケースである。

## 最後のセクション

このセクションは最後のセクション。日本語コンテンツのトークン推定もテストする。
セマンティック検索では、日本語と英語が混在するドキュメントを正確に処理する
必要がある。intfloat/multilingual-e5-large モデルは 100 以上の言語に対応しており、
日本語のドキュメントでも高い精度でベクトル表現を生成できる。

テスト用のダミーテキスト:
- 設計書の整合性チェック
- ファイル参照の生存確認
- ルール・スキルの鮮度チェック
- ADR のステータス検証
- クロスリファレンスの妥当性確認

## セマンティック検索の仕組み

sync テスト用の追記。

セマンティック検索は、従来のキーワード検索（Grep）とは異なり、テキストの「意味」を
ベクトル（数値の配列）に変換して類似度を計算する手法である。たとえば「認証」と
「ログイン」は文字列としては異なるが、意味的には近いため、セマンティック検索では
関連ドキュメントとして検出できる。qdrant-indexer では intfloat/multilingual-e5-large
モデルを使用し、1024次元のベクトル空間にドキュメントをマッピングしている。
このモデルは100以上の言語に対応しており、日本語と英語が混在するドキュメントでも
高い精度でベクトル表現を生成できる。チャンク分割は h2 セクション単位で行い、
各チャンクに「ファイル名 > h1 > h2」形式のプレフィックスを付与することで、
検索結果の文脈を保持している。
