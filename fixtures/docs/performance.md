# パフォーマンスチューニング

qdrant-indexer のパフォーマンス改善に関するガイドライン。

## バッチサイズの調整

大量のドキュメントをインデックスする場合、バッチサイズがスループットに大きく影響する。
デフォルトのバッチサイズは 64 だが、メモリに余裕がある場合は 128 や 256 に増やすことで
処理速度を向上できる。GPU を使用する場合は VRAM の容量に応じて調整する。

## メモリ使用量の最適化

embedding モデルのロード時に約 1.5GB のメモリを消費する。
コンテナ環境ではメモリ制限を 4GB 以上に設定することを推奨する。
OOM が発生する場合はバッチサイズを小さくするか、モデルの量子化を検討する。
