# バッチ処理設計ドキュメント

このドキュメントは qdrant-indexer のバッチ処理機能に関する設計資料である。

## 概要

qdrant-indexer v0.1.0 では、大量のドキュメントを処理する際の
メモリ効率を改善するためにバッチ処理パイプラインを導入した。

従来の処理フロー:
1. 全ファイルをチャンク分割
2. 全チャンクを一括ベクトル化
3. 全ベクトルを一括で Qdrant に投入

新しい処理フロー:
1. 全ファイルをチャンク分割
2. チャンクを 32 件ずつバッチに分割
3. バッチごとにベクトル化 → Qdrant 投入を繰り返す

## アーキテクチャ

バッチ処理は CLI レイヤー（cli.py）で実装されている。
Embedder と QdrantIndexer の API は変更せず、呼び出し側で
バッチ分割を行う設計とした。これにより既存のユニットテストへの
影響を最小限に抑えている。

### コンポーネント間の関係

- cli.py: バッチ分割のオーケストレーション
- embedder.py: バッチ単位のベクトル生成（API 変更なし）
- indexer.py: バッチ単位の upsert（API 変更なし）

## 設定

現在、バッチサイズはモジュール定数 `_EMBED_BATCH_SIZE = 32` として
ハードコードされている。将来的には設定ファイルから変更可能にする予定。
